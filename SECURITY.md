## Supported Versions

Fluttrap is under active development. For security reasons, **only the latest released version** is supported.

Always upgrade to the most recent version to receive security fixes and improvements.

---

## Reporting a Vulnerability

We strongly encourage **responsible disclosure**.

### How to report
- Please **do not open public issues** for security vulnerabilities.
- Submit a private report using GitHubâ€™s **Private Vulnerability Reporting** feature  
  **or**
- Email us directly at: contact@belovance.com

### What to include
- Clear description of the issue
- Steps to reproduce
- Potential impact and severity
- Any proof-of-concept or mitigation ideas

We aim to acknowledge all valid reports within **48 hours**.

---

## Disclosure & Resolution Process

Once a vulnerability is confirmed:

1. The issue is reviewed and severity is assessed
2. A fix is developed, tested, and validated
3. A patched release is published
4. A security advisory is issued when appropriate

Our goal is to resolve security issues responsibly, efficiently, and transparently.

---

## Scope

Fluttrap is a **UI framework** and does not directly handle:
- Authentication
- User data persistence
- Networking or API communication
- Payments or sensitive user information

However, the following are considered **in scope**:
- UI-level security risks
- Unsafe defaults or configurations
- Dependency vulnerabilities
- Code patterns that may introduce security concerns

---

## Dependency Security

Fluttrap relies on trusted packages from the Flutter ecosystem. Dependencies are:
- Actively monitored
- Regularly updated
- Reviewed for known vulnerabilities

---

## Best Practices for Users

We recommend all users:
- Keep Fluttrap and Flutter SDK up to date
- Follow Flutter security guidelines
- Regularly audit project dependencies
- Avoid modifying core framework internals
